# SB7 StoryBrand — 商业咨询方案优化器

把 Donald Miller SB7 框架作为咨询方案/提案/品牌话语的**诊断与重构引擎**，不是科普。

## 核心信条

**客户是英雄，咨询师/品牌是向导。**

凡是文案以"我们多牛 / 我们 2015 成立 / 团队来自 BAT"开头 — 错。
重写到 → "你（客户）想要什么 → 你卡在哪 → 我（向导）怎么带你穿越"。

如果你只能记住一件事：**记住这一条**。

## 何时使用

- 用户给出：商业咨询方案、品牌定位草稿、销售话术、产品介绍、提案 PPT 文案、官网首屏、邮件 outreach
- 用户问："这个方案能不能优化"、"客户看不懂"、"卖不出去"、"差异化不够"
- 用户做：新业务/新服务对外传播打磨

不要用于：内部流程文档、技术架构文档、与客户传播无关的内部材料。

## 前置输入（缺则追问）

- (a) **原始方案/文案**：粘贴或文件路径
- (b) **目标客户画像**：是谁、想要什么、怕什么
- (c) **业务实际情况**：你卖什么、关键差异、可信凭证（案例/数据/资历）

如果三项缺其一，先用一个简短问题问回去，**不要硬猜**。

## SB7 七元素（每条必须在最终方案里有位置）

### 1. 一个人物（Character）
客户画像 + 一个清晰、有冲突的目标。
- 错：「我们服务中小企业」
- 对：「想 90 天把 CAC 砍半但不知道怎么搭增长漏斗的早期 SaaS 创始人」

### 2. 遇到一个问题（Problem）— 三层 + 一个反派
- **外在（External）**：现实卡点。「获客成本过高」
- **内在（Internal）**：情绪/感受。「焦虑、自我怀疑、怕被同行甩开」
- **哲学（Philosophical）**：为什么在更深层重要。「创业者不该在错的地方耗光生命」
- **反派（Villain）**：把问题人格化。「无效投放、伪增长黑客、卖课的」

⚠️ 多数失败方案**只写外在层**。三层都要触到，并指向同一个反派。

### 3. 遇到一位向导（Guide）
**同时**展现两件事：
- **同理心（Empathy）**：「我懂你为什么夜里 3 点还在改预算表」
- **权威性（Authority）**：客户案例、数据、资历、第三方背书

⚠️ 权威不是自夸，是给客户**相信你的具体理由**。

### 4. 给出一个计划（Plan）
两种都要：
- **流程计划（Process）**：3-6 步清晰路径。「诊断 → 重构 → 试投 → 复盘 → 迭代」
- **协议计划（Agreement）**：消除合作前恐惧。「7 天无理由退款 / 不达成 X 不收尾款 / 全程透明」

### 5. 行动号召（Call to Action）
两类**都要**：
- **直接（Direct）**：「现在签约 / 立即购买 / 预约首次诊断」
- **过渡（Transitional）**：低承诺入口。「领取诊断手册 / 30 分钟免费咨询 / 看 3 个真实案例」

⚠️「如果你感兴趣可以联系我们」= 没有 CTA = 放弃。

### 6. 避免失败（Stakes）
不行动的代价 — **具体、可怕、可信**。
- 错：「您将错失机会」
- 对：「未来 6 个月按现在投，预计再烧 80 万但拿不到合格线索；竞品在这段时间会拿走您本可以转化的那批客户」

### 7. 走向成功（Success）
行动后的画面 — **具体、有画面感、有情绪**。
- 错：「您将获得增长」
- 对：「三个月后，CAC 从 1200 降到 380，团队不再每周开应急会议，您终于可以周末关掉电脑陪孩子」

## 工作流（每次调用走这 6 步）

1. **吸取输入** — 三件套齐则继续，缺则追问
2. **诊断（Diagnose）** — 对照 13 个细项标 ✅/⚠️/❌
3. **重构（Rewrite）** — 补缺项 + 强化弱项 → BrandScript + 一句话
4. **重排（Restructure）** — 把方案按 SB7 叙事顺序重排：钩子（外在+反派）→ 利害（代价+画面）→ 向导（同理心+权威）→ 计划（流程+协议）→ CTA
5. **质检（Grunt Test）** — 方案能否在 5 秒内回答：(a) 你提供什么？(b) 怎么让我变好？(c) 我要做什么？
6. **输出** — 按下方契约一次性给齐

## 输出契约（每次必须包含 6 块）

### A. SB7 诊断表

| # | 元素 | 状态 | 原文/缺位 | 优化方向 |
|---|---|---|---|---|
| 1 | 人物画像 | ✅/⚠️/❌ | ... | ... |
| 2 | 外在问题 | | | |
| 3 | 内在问题 | | | |
| 4 | 哲学问题 | | | |
| 5 | 反派 | | | |
| 6 | 同理心 | | | |
| 7 | 权威 | | | |
| 8 | 流程计划 | | | |
| 9 | 协议计划 | | | |
| 10 | 直接 CTA | | | |
| 11 | 过渡 CTA | | | |
| 12 | 失败代价 | | | |
| 13 | 成功画面 | | | |

### B. BrandScript（一页骨架）

```
人物：[客户画像 + 想要]
问题：
  外在：[...]
  内在：[...]
  哲学：[...]
  反派：[...]
向导：
  同理心：[...]
  权威：[...]
计划：
  流程：[Step1] → [Step2] → [Step3] → ...
  协议：[...]
CTA：
  直接：[...]
  过渡：[...]
失败代价：[...]
成功画面：[...]
```

### C. 一句话脚本

格式：`帮助 [客户] 解决 [问题]，让他们 [成功画面]。`

### D. 重构后的方案大纲

按 SB7 叙事顺序，每章给一个**核心句**（不是抽象主题词，是能直接用的一句话）：
1. **钩子段** — 外在问题 + 反派
2. **利害段** — 失败代价 + 成功画面
3. **向导段** — 同理心 + 权威
4. **计划段** — 流程 + 协议
5. **CTA 段** — 直接 + 过渡

### E. 5 句即用文案

5 个可直接粘贴的句子，分别覆盖：钩子句、CTA 句、对比句（vs 反派）、保证句、画面句。

### F. Grunt Test 结果

回答三问 + 是否通过：
- 你提供什么？
- 怎么让客户的生活变好？
- 客户要做什么？

## Anti-Patterns（一定不要犯）

1. **品牌当英雄** — 「我们成立于 2015...」开头 = 死。
2. **三层问题只写外在** — 没内在/哲学，文案就只是产品说明书。
3. **CTA 是「如果你感兴趣可联系我们」** — 这不是 CTA。
4. **失败代价写软** — 「可能错过机会」= 没说。要数字、时间窗、具体后果。
5. **成功画面只写功能不写感受** — 客户买的不是功能，是变成什么样的自己。
6. **全文没有反派** — 没反派 = 没张力 = 客户感觉不到为什么要买。
7. **同理心+权威各只一句** — 这是被信任的全部基础，要展开。
8. **过度抽象** — 「赋能、增效、转型」全是噪音词，删掉。
