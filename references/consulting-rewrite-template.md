# 咨询方案 SB7 重构模板

把任意咨询方案/提案 PPT 重排成 SB7 叙事顺序的标准骨架。每一节给出**位置 + 长度 + 核心句结构 + 反例 + 正例**。

---

## 整体结构（5 段叙事）

```
┌────────────────────────────────────────────┐
│ §1 钩子段（外在问题 + 反派）           ~10% │
├────────────────────────────────────────────┤
│ §2 利害段（失败代价 + 成功画面）       ~15% │
├────────────────────────────────────────────┤
│ §3 向导段（同理心 + 权威）             ~20% │
├────────────────────────────────────────────┤
│ §4 计划段（流程 + 协议）               ~30% │
├────────────────────────────────────────────┤
│ §5 CTA 段（直接 + 过渡）               ~15% │
├────────────────────────────────────────────┤
│ 附录（FAQ / 案例集 / 团队）            ~10% │
└────────────────────────────────────────────┘
```

⚠️ 关键：**前 30% 决定客户读不读完**。所以必须在前 30% 解决「客户是谁、卡在哪、不解决会怎样、解决后是什么样」。

---

## §1 钩子段（外在问题 + 反派）

**目的**：让客户在第一段里点头说"对，这就是我"。

**长度**：1-2 段，最多 5-7 行。

**结构**：
```
[一句话画出客户当前处境的具体场景]
[把外在问题人格化为反派]
[暗示这不是客户能力问题]
```

**反例**：
> 在数字化时代，企业获客面临前所未有的挑战，转化率持续下滑，已成为困扰众多创业者的难题...

（抽象、宏观、所有人都能写、客户读完毫无感觉）

**正例**：
> 你已经在投放上烧了 6 个月，每个月 50 万预算，但合格线索还是不到 30 个。
> 不是你不努力 — 是那些把"增长黑客"包装成 99 元课程的人，用"私域、漏斗、ABM"几个词语，让你以为问题是你掌握得不够多。
> 真正的问题是：你的话语层从来没被设计过。

---

## §2 利害段（失败代价 + 成功画面）

**目的**：让客户在做决策前就感到「不行动会失去什么、行动后会得到什么」。

**长度**：2 段（一段代价，一段画面），各 4-6 行。

**结构**：

**§2a 失败代价**
```
[时间窗] + [按现状推算的具体后果] + [谁会替代你/吃掉你的位置]
```

**§2b 成功画面**
```
[3-6 个月后的具体场景] + [可验证的数字] + [情绪状态]
```

**反例**：
> 不及时优化将错失市场机会。优化后将获得显著增长，提升竞争力。

**正例**：
> **如果继续按现状跑 6 个月**：再烧 300 万预算，CAC 维持在 1500 以上，合格线索池仍然不够 A 轮 DD 的需求。同期已有 2 家竞品从你的目标客户名单里签下了 18 家。
>
> **如果重构话语层**：90 天后 CAC 降到 600 以下，每月稳定 80+ 合格线索，团队不再每周开应急会议。下次和投资人聊增长时，你拿出来的不是借口，是一份可复制的剧本。

---

## §3 向导段（同理心 + 权威）

**目的**：客户已经被钩住、感到了利害，现在要回答"为什么是你"。

**长度**：3-4 段。

**结构**：

**§3a 同理心**（1 段）
```
[描述客户内心独白，让客户觉得"卧槽你怎么知道"]
```

**§3b 权威**（2-3 段）
```
[具体客户结果：X 客户，Y 数字，Z 时间]
[第三方背书 / 数据规模 / 方法论沉淀]
[相关从业经验 — 但只用一句]
```

**反例**：
> 我们是一支资深团队，拥有丰富的增长经验，曾服务多家知名企业，致力于为客户提供卓越的解决方案。

**正例**：
> 我知道你已经看完 50 篇增长黑客文章，每读一篇都更焦虑而不是更清楚。我也知道你最怕的不是没人懂这事，是请来的"专家"复制粘贴方法论，最后烧完预算告诉你"市场环境变了"。
>
> 过去 18 个月我们帮 23 家早期 SaaS 重构话语层，平均把 CAC 砍掉 47%。其中：[客户 A] 90 天 CAC 从 ¥1,800 降到 ¥620，[客户 B] 完成 A 轮，[客户 C] 在小红书自然增长 8 倍。所有案例数据可查。
>
> 我们的诊断模型已经跑过 200+ 早期 SaaS，方法论由前阿里增长团队负责人 [姓名] 主导设计，曾被 [媒体/机构] 报道。

---

## §4 计划段（流程 + 协议）

**目的**：减少客户**承诺前的认知摩擦**。让"接下来怎么走"变得清晰、可控、低风险。

**长度**：方案最长的一段。流程 1 页，协议 1 页。

**结构**：

**§4a 流程计划**（编号 + 动词 + 产物）
```
第 1 周：[动作] → 产出 [可交付物]
第 2 周：[动作] → 产出 [可交付物]
...
```

**§4b 协议计划**（每条对应一个客户内心的"不"）
```
- 不用怕被坑：[退款条款]
- 不用怕没结果：[绑定指标条款]
- 不用怕被绑：[终止条款]
- 不用怕信息黑箱：[透明条款]
- 不用怕队伍不专业：[人员条款]
```

**反例**：
> 我们的服务流程包括：调研、方案、执行、复盘四个阶段，全程提供专业服务。

**正例**：
> **流程**
> - **第 1 周** — 增长漏斗诊断 → 产出《卡点报告》（含 12 个量化诊断维度）
> - **第 2 周** — 话语层重构 → 产出 BrandScript + 5 套对照投放素材
> - **第 3-4 周** — 试投 + 数据回收 → 产出试投复盘报告
> - **第 5 周** — 定稿 → 产出可复制的投放剧本（含 SOP）
> - **第 6 周起** — 双周复盘机制（共 8 次）→ 持续迭代
>
> **协议**
> - 不达成约定的 CAC 降幅 30%，全额退款
> - 全程双周对齐，所有数据透明，您可随时终止合作
> - 每季度只接 4 个客户，确保深度服务
> - 主咨询师全程在场，不外包给实习生
> - 服务结束后 30 天内的紧急问题免费咨询

## §5 CTA 段（直接 + 过渡）

**目的**：给所有人一个出口 — 准备好的人有直接 CTA，没准备好的人有过渡 CTA。

**长度**：3-5 行。

**结构**：
```
[直接 CTA — 大、粗、明确动作]
[过渡 CTA — 低承诺入口]
[最后一句重申失败代价或时间窗]
```

**反例**：
> 期待与您合作！如有兴趣可联系我们：xxx@xxx.com

**正例**：
> **现在做的事：**
> 👉 [预约 30 分钟首次诊断（免费）](#) — 我们直接看你的数据，给一份初步卡点判断
> 📥 [下载《SaaS 增长漏斗诊断手册》PDF](#) — 没准备好沟通？先用这本自检
>
> 本季度还剩 2 个名额。下季度起服务费上涨 20%。

---

## 附录段

可选放：
- FAQ（5-7 条）
- 完整客户案例 1-2 个（每个 1 页）
- 团队/创始人介绍（**只用一段**，不要喧宾夺主）
- 媒体/机构背书图

---

## 检查清单（重构完最后跑一遍）

- [ ] 前 30% 是否包含「客户是谁 + 卡在哪 + 不行动后果 + 行动后画面」
- [ ] 是否在前 3 屏出现至少 1 个直接 CTA
- [ ] 整篇是否找到了**一个反派**？
- [ ] 失败代价是否含「时间窗 + 数字 + 谁替代你」三件套？
- [ ] 成功画面是否含「数字 + 场景 + 情绪」三件套？
- [ ] 流程计划是否每步都有**动词 + 可交付物**？
- [ ] 协议计划是否回答了至少 3 个客户内心的"不"？
- [ ] 全文是否删干净了「赋能 / 增效 / 转型 / 卓越 / 专业」等噪音词？
- [ ] Grunt Test：5 秒内能回答「你提供什么 / 怎么让我变好 / 我要做什么」？
